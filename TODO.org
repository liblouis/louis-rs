- [X] Word/group-sign ("lower"/part-word sign) contraction not applied
  when the word carries a suffix, apostrophe, or is part of a longer
  compound, even though the same sign fires correctly when the word
  stands alone
  - root cause found and fixed: nearly all forward-direction UEB
    word/group-signs ("would", "little", "it", "was", ...) are defined
    only via ~match~ rules gated on the ~^~ boundary attribute (see
    above), so none of them fired at all, for any input shape,
    lowercase or not — not a trie precedence/longest-match issue as
    first suspected
  - confirmed fixed for lowercase input: "would", "little", "it",
    "was", "also" etc. now all contract correctly forward
  - [ ] NEW, more specific remaining gap: capitalized input still
    doesn't contract ("Would"/"Little"/"Also" still spell out fully) —
    the ~match~ opcode's literal-string comparison
    (~Instruction::Char~ in regexp.rs) does exact case-sensitive
    equality with no case-folding, so any capitalized occurrence of a
    word-sign falls through. Given a full-word dictionary test is
    heavily weighted toward capitalized proper nouns and
    sentence-initial words, this is likely now the largest remaining
    contributor to ~en-ueb-g2-dictionary_harness.yaml~'s failures

- [ ] Normalize input text to a canonical Unicode form
  - Input is currently normalized to NFC at the start of the
    translation pipeline. This fixes tables that define rules for
    precomposed characters (e.g. pt-pt-g1.utb), but breaks tables
    that define rules for standalone combining characters (IPA, Hebrew,
    Yiddish). NFD normalization is a promising alternative — it

    precomposed rules still match via longer trie paths.

  applied when the word carries a suffix, apostrophe, or is part of a
  longer compound, even though the same sign fires correctly when the
  word stands alone

    in full instead of contracting it; "'twould", "would've" etc. lose
    the "would" wordsign; "it"/"was"/"also" lose their UEB short forms
    when combined with other punctuation/context

    ~en-ueb-g2-dictionary_harness.yaml~ alone has ~16,600 failing
    tests from this (56% of all failing tests suite-wide); also hits
    ~afr-za-g2.yaml~ (Afrikaans groupsigns) and
    ~hu-hu-g1_dictionary_special_consonants.yaml~ (gy/ggy digraph
    precedence)
  - likely a trie precedence/longest-match bug for lower/part-word
    signs

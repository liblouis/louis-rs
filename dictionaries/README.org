* Hyphenation dictionaries

This directory holds ~.bincode~ files used by the ~hyphen~/~nocross~
opcode handling in ~src/translator/table/primary.rs~. They are
Knuth–Liang hyphenation pattern dictionaries in the binary format
produced by the [[https://docs.rs/hyphenation][hyphenation]] crate,
which louis-rs uses at runtime (via ~Standard::any_from_reader~) to
decide where a word may be split across braille lines.

They are *not* built from this repository. They are generated in a
sibling checkout of the [[https://github.com/tapeinosyne/hyphenation][hyphenation]]
crate from pattern files copied out of liblouis's ~tables/~ directory,
and the resulting ~.bincode~ is copied back here by hand. There is
currently no script for this — see "Why this is manual" below.

** How table files reference these dictionaries

A liblouis table pulls in a hyphenation dictionary with a normal
~include~ opcode, e.g.:

#+begin_example
include dictionaries/de-g2-core-patterns.dic
#+end_example

~src/parser.rs~ (~expand_include~) special-cases any ~include~ whose
target ends in ~.dic~: it swaps the extension to ~.bincode~ and looks
up that exact filename on ~LOUIS_TABLE_PATH~. *The bincode file in
this directory must therefore be named identically to the liblouis
~.dic~ file it replaces, with the extension changed* — nothing else
about the original filename may change.

** Which liblouis files these come from

liblouis's ~tables/~ directory has two unrelated families of ~.dic~
files. Only the second kind belongs here:

- ~hyph_<lang>_<REGION>.dic~ (e.g. ~hyph_de_DE.dic~) — plain-text
  hyphenation patterns for printed text. Not used by louis-rs;
  unrelated to this directory.
- A handful of *braille* hyphenation-pattern dictionaries, used to
  decide where contracted/uncontracted braille may be split. These
  are the ones converted here:

  | liblouis source                | hyphenation crate ~Language~ | code               | bincode here                | minima (left, right)        |
  |---------------------------------+-------------------------------+--------------------+------------------------------+------------------------------|
  | ~da-dk-g2.dic~                 | ~DanishBrailleGrade2~        | ~da-braille-g2~    | ~da-dk-g2.bincode~          | (1, 1) — from header        |
  | ~de-g1-core-patterns.dic~      | ~GermanBrailleGrade1~        | ~de-ch-braille-g1~ | ~de-g1-core-patterns.bincode~ | (1, 1) — default, no header |
  | ~de-g2-core-patterns.dic~      | ~GermanBrailleGrade2~        | ~de-ch-braille-g2~ | ~de-g2-core-patterns.bincode~ | (1, 1) — default, no header |
  | ~en-g3-patterns.dic~           | ~EnglishBrailleGrade3~       | ~en-braille-g3~    | ~en-g3-patterns.bincode~    | (1, 1) — from header        |

(~hyphenation_commons/src/language.rs~ and ~build.rs~ in the
~hyphenation~ checkout are where these ~Language~ variants and their
~(left, right)~ minima are declared. See "Choosing the minima bounds"
below for what that tuple means and how the values above were
verified.)

There is also a ~Language::Test~ (~test-braille~) entry in the
~hyphenation~ checkout, built from a copy of liblouis's own
~tests/yaml/hyphenation.dic~ fixture. It is *not* currently copied
into this directory or referenced by any table — it exists only as a
ready-to-use replacement should a future test need the fixture that
used to live at ~dictionaries/hyphenation.bincode~ (removed in
~ed55f45~ because it no longer deserialized — see below for why).

** Adding or updating one of these dictionaries

1. In a checkout of the ~hyphenation~ crate, copy the source file
   verbatim:
   #+begin_src shell
   cp path/to/liblouis/tables/<name>.dic patterns/hyph-<code>.pat.txt
   #+end_src
2. Register the language in *both*:
   - ~hyphenation_commons/src/language.rs~ — add a ~fiant_linguae!~
     entry: ~VariantName, (left_min, right_min), "<code>";~ (bounds
     come from the source file's ~LEFTHYPHENMIN~/~RIGHTHYPHENMIN~
     header, or ~(1, 1)~ if absent).
   - ~build.rs~ — add the same variant name to the ~_std_langs~
     vector.
3. Build every dictionary from patterns (writes only under
   ~target/~, never touches the crate's own committed
   ~dictionaries/~):
   #+begin_src shell
   cargo build --features build_dictionaries
   find target -type d -name dictionaries   # locate the OUT_DIR copy
   #+end_src
4. Copy ~<code>.standard.bincode~ from that output directory into
   this directory, renamed to match the original liblouis ~.dic~
   filename (see table above), and run ~cargo test~ here to confirm
   nothing regressed.

** Choosing the minima bounds

Each ~Language~ entry carries a ~(left, right)~ pair — the minimum
number of characters that must remain before/after any break (TeX's
~\lefthyphenmin~/~\righthyphenmin~). It's enforced by the
~hyphenation~ crate itself in ~Hyphenator::boundaries()~; get it
wrong and you either lose real break opportunities near the edges of
short words (too high) or allow breaks the source dictionary never
intended (too low, though in practice these dictionaries rarely
define patterns that close to an edge anyway).

Some ~.dic~ files declare this explicitly via a ~LEFTHYPHENMIN N~ /
~RIGHTHYPHENMIN N~ header (a vestige of the libhyphen/OpenOffice
hyphenation-dictionary format some of these files were derived from).
*liblouis's own table compiler does not implement this directive at
all* — ~compileHyphenation()~ in ~compileTranslationTable.c~ silently
absorbs the header line as an inert, all-zero pattern (it isn't a
~#~/~%~ comment, but it contains no digits either, so it never
produces a break). So the header carries no weight for liblouis
itself, but it /is/ the original pattern author's stated intent, and
it's the right value for us to honor since we're the ones enforcing
it.

When a ~.dic~ file has no such header (both German ones here),
there's no signal to go on; ~(1, 1)~ — the least restrictive choice —
is used as the default and hasn't been contradicted by anything found
so far.

*Don't guess by copying the entry above it in ~language.rs~.* That's
literally how ~DanishBrailleGrade2~ ended up wrong for a while: it
was registered as ~(2, 2)~, an exact copy of the plain ~Danish~ entry
directly above it, even though ~da-dk-g2.dic~ explicitly declares
~LEFTHYPHENMIN 1~ / ~RIGHTHYPHENMIN 1~ (the same bounds as liblouis's
own ~hyph_da_DK.dic~, which ~da-dk-g2.dic~ was derived from). Always
check the source file's own header first.

** The one non-obvious gotcha

~Language~ is serialized by *enum position*, not by its ~code~
string. Inserting a new variant anywhere before an existing one in
~fiant_linguae!~ shifts the serialized discriminant of every variant
that follows it in the list — which silently invalidates every
~.bincode~ built from those later variants, even though their source
patterns never changed. (This is exactly how the old
~dictionaries/hyphenation.bincode~ fixture broke, and it happened
again to ~de-g1-core-patterns.bincode~ / ~de-g2-core-patterns.bincode~
when ~EnglishBrailleGrade3~ was inserted before them.)

*Rule of thumb: whenever a language is added to ~language.rs~,
rebuild and recopy every dictionary in this directory, not just the
new one — then diff against what's currently committed to be sure
nothing else silently changed.*

** Why this is manual

These tables change upstream roughly once every few years, so a
dedicated sync script hasn't been worth building yet. If that
changes, the natural fix is twofold: make ~Language~'s
(de)serialization keyed on ~code~ instead of enum position (removing
the gotcha above), and add a small script that performs steps 1–4
given a liblouis checkout path and a ~.dic~ filename.
